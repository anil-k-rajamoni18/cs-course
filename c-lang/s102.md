## Operators 

- In C (and most programming languages), operators are symbols or tokens that tell the compiler to perform specific operations on one or more operands (values or variables).


### ‚öôÔ∏è 1Ô∏è‚É£ Arithmetic Operators

- Arithmetic operators perform basic mathematical calculations on numeric operands.
- They are fundamental to expression evaluation and numeric manipulation.

**üí° Where:**
- Used in almost every computation ‚Äî from simple sums to algorithmic logic, loops, and conditional processing.

| Operator | Operation      | Example | Result |
| -------- | -------------- | ------- | ------ |
| `+`      | Addition       | `5 + 3` | `8`    |
| `-`      | Subtraction    | `5 - 3` | `2`    |
| `*`      | Multiplication | `5 * 3` | `15`   |
| `/`      | Division       | `5 / 2` | `2`    |
| `%`      | Modulus        | `5 % 2` | `1`    |


**NOTE**
- Division between integers truncates the decimal part.
```c
7 / 2 ‚Üí 3
7.0 / 2 ‚Üí 3.5
```
- % works only on integers.
- Use casting for floating operations:
```c
float result = (float)a / b;
```

---
### ‚öôÔ∏è 2Ô∏è‚É£ Relational Operators
- They compare two values and return a Boolean result ‚Äî 1 (true) or 0 (false).

**üí° Where:**
- Used in decision-making statements (if, while, for) to control program flow.

| Operator | Meaning               | Example    | Result |
| -------- | --------------------- | ---------- | ------ |
| `<`      | Less than             | `5 < 10`   | 1      |
| `>`      | Greater than          | `5 > 10`   | 0      |
| `<=`     | Less than or equal    | `10 <= 10` | 1      |
| `>=`     | Greater than or equal | `20 >= 10` | 1      |
| `==`     | Equal to              | `5 == 5`   | 1      |
| `!=`     | Not equal to          | `5 != 3`   | 1      |


**‚ö†Ô∏è Common Pitfall:**
- Using = instead of == inside conditions:
```c
if (a = b)  // assigns, doesn‚Äôt compare!
```
----
### ‚öôÔ∏è 3Ô∏è‚É£ Logical Operators
- Combine or negate conditions to form complex logical expressions.

**üí° Where:**
- Used for control flow in multi-condition logic.

| Operator | Description | Example  | Result     |    |   |    |   |
| -------- | ----------- | -------- | ---------- | -- | - | -- | - |
| `&&`     | Logical AND | `1 && 0` | 0          |    |   |    |   |
| `        |             | `        | Logical OR | `1 |   | 0` | 1 |
| `!`      | Logical NOT | `!1`     | 0          |    |   |    |   |

**üß† Short-circuiting:**
- `a && b ‚Üí stops if a is false.`
- `a || b ‚Üí stops if a is true.`
- This avoids unnecessary computation or invalid memory access.


```c
if (ptr && *ptr == 'A')  // Safe: *ptr not checked if ptr is NULL
```

---
### ‚öôÔ∏è 4Ô∏è‚É£ Bitwise Operators
- Operate on individual bits of integer data.

**üí° Where:**
- Used in low-level programming, flags, masks, and embedded systems.

| Operator | Description | Example  | Result (Binary)          |    |            |
| -------- | ----------- | -------- | ------------------------ | -- | ---------- |
| `&`      | AND         | `5 & 3`  | `0101 & 0011 = 0001 (1)` |    |            |
| `        | `           | OR       | `5                       | 3` | `0111 (7)` |
| `^`      | XOR         | `5 ^ 3`  | `0110 (6)`               |    |            |
| `~`      | NOT         | `~5`     | Inverts bits             |    |            |
| `<<`     | Left shift  | `5 << 1` | `10`                     |    |            |
| `>>`     | Right shift | `5 >> 1` | `2`                      |    |            |

**Example**
```c
int x = 8;   // 1000
x = x >> 1;  // 0100 ‚Üí 4
x = x << 2;  // 10000 ‚Üí 16
```

----
### ‚öôÔ∏è 5Ô∏è‚É£ Increment & Decrement Quirks
- Increase (++) or decrease (--) the value of a variable by 1.

**üí° Where:**
- Widely used in loops, counters, and iterative logic.

| Form  | Description    | Example       | Behavior |
| ----- | -------------- | ------------- | -------- |
| `++x` | Pre-increment  | `x=5; y=++x;` | x=6, y=6 |
| `x++` | Post-increment | `x=5; y=x++;` | x=6, y=5 |

**Example:**
```c
int x = 5;
printf("%d", x++ + ++x);
```

---
### ‚öôÔ∏è 6Ô∏è‚É£ Conditional / Ternary Operator
- Compact form of if-else that returns a value.

**üí° Where:**
- Saves lines and allows inline decisions in expressions.

**Syntax:**
```c
(condition) ? expression_if_true : expression_if_false;
```

```c
int a = 10, b = 20;
int max = (a > b) ? a : b;  // Same as if(a>b) max=a; else max=b;
```

```c
int largest = (a > b) ? ((a > c) ? a : c)
                      : ((b > c) ? b : c);
```

---
### ‚öôÔ∏è 7Ô∏è‚É£ Assignment Chaining
- Assigning multiple variables in one statement.

**üí° Where:**
- Efficient shorthand initialization when all need the same value.

```c
int a, b, c;
a = b = c = 10;
```

**‚öôÔ∏è Internal Working**
- Assignment is right-to-left.
- c = 10 returns 10, which is assigned to b, then a.

---
### ‚öôÔ∏è 8Ô∏è‚É£ Operator Precedence & Associativity
- Defines the order in which operators are evaluated.

**üí° Where:**
- To ensure predictable expression outcomes.

| Precedence     | Operator(s)        | Associativity |   |              |
| -------------- | ------------------ | ------------- | - | ------------ |
| Highest        | `()` `[]` `->` `.` | Left ‚Üí Right  |   |              |
| Unary          | `++` `--` `!` `~`  | Right ‚Üí Left  |   |              |
| Multiplicative | `* / %`            | Left ‚Üí Right  |   |              |
| Additive       | `+ -`              | Left ‚Üí Right  |   |              |
| Shift          | `<< >>`            | Left ‚Üí Right  |   |              |
| Relational     | `< <= > >=`        | Left ‚Üí Right  |   |              |
| Equality       | `== !=`            | Left ‚Üí Right  |   |              |
| Logical AND    | `&&`               | Left ‚Üí Right  |   |              |
| Logical OR     | `                  |               | ` | Left ‚Üí Right |
| Conditional    | `?:`               | Right ‚Üí Left  |   |              |
| Assignment     | `= += -= *=`       | Right ‚Üí Left  |   |              |
| Comma          | `,`                | Left ‚Üí Right  |   |              |


**‚öôÔ∏è Example:**
```c
int x = 10 + 5 * 2;   // 5*2 first ‚Üí 20
int y = (10 + 5) * 2; // parentheses change order
```

---

### ‚öôÔ∏è 9Ô∏è‚É£ Short-Circuit Behavior
- In logical operations, evaluation stops as soon as the result is known.

**üí° Why:**
- Saves computation and avoids invalid operations (e.g., null pointer access).

```text
| Operator | Stops When            | Example                           |                    
| -------- | --------------------- | --------------------------------- | 
| `&&`     | Left operand is false | 0 && func() ‚Üí func() not called   |
| `||`     | Left operand is true  | 1 || func() ‚Üí func() not called   |

```

```c
int a = 0, b = 10;
if (a && ++b) {}  // b not incremented
printf("%d", b);  // 10
```

---
### ‚öôÔ∏è üîü Type Effects in Expressions
- When operands have different data types, C automatically promotes or converts them before evaluation.

**üí° Where:**
- To prevent loss of precision or undefined results in mixed-type operations.

**üî† Type Promotion Rules:**
1. char and short ‚Üí promoted to int
2. If one operand is float or double, others are converted to it
3. Unsigned types dominate signed if both have the same rank

```c
int a = 5;
float b = 2;
float c = a / b;   // a promoted ‚Üí 2.5
```

```c
char x = 120;
int y = 10;
printf("%d", x + y);  // x promoted to int
```

**‚ö†Ô∏è Explicit Casting:**
- Use when implicit promotion is unsafe:
```c
int x = 5, y = 2;
printf("%f", (float)x / y); // 2.5
```

# üß© Conditional Statements

**üîπ What are Conditional Statements?**
- Conditional statements control the flow of a program based on whether a condition is true or false.
- They help the program make decisions dynamically.
- Think of them as:
    - "If something is true ‚Äî do this, otherwise ‚Äî do something else."

**üîπ Why We Need Them**
- To make decisions in programs
- To perform different actions based on conditions
- To implement logic like validations, comparisons, or options

```c
if (marks >= 50)
    printf("Pass\n");
else
    printf("Fail\n");
```

### üî∏ if Statement
- Used when you want to perform an action only if a condition is true.
- syntax
```c
if (condition) {
    // code if condition is true
}
```

```c
int age = 20;
if (age >= 18) {
    printf("You can vote!\n");
}
```

### üî∏ if-else Statement
- Used when there are two possible outcomes ‚Äî one if true, one if false.
```c
int num = 5;
if (num % 2 == 0)
    printf("Even\n");
else
    printf("Odd\n");
```

### üî∏ if-else if-else Ladder
- Used when there are multiple conditions to check in sequence.
- syntax
```c
if (condition1)
    statement1;
else if (condition2)
    statement2;
else
    statement3;
```

```c
int marks = 75;
if (marks >= 90)
    printf("Grade A\n");
else if (marks >= 75)
    printf("Grade B\n");
else if (marks >= 50)
    printf("Grade C\n");
else
    printf("Fail\n");
```

---
### üß© Nested Conditionals
- A nested conditional means one if inside another.
- Used when a decision depends on another decision.

**Example 1: Nested if**
```c
int age = 25;
char citizen = 'Y';

if (age >= 18) {
    if (citizen == 'Y')
        printf("Eligible to vote\n");
    else
        printf("Not a citizen\n");
} else {
    printf("Underage\n");
}
```

**Example 2: Nested if-else chain**
```c
int a = 10, b = 20, c = 15;

if (a > b) {
    if (a > c)
        printf("a is largest\n");
    else
        printf("c is largest\n");
} else {
    if (b > c)
        printf("b is largest\n");
    else
        printf("c is largest\n");
}
```

---
### switch Statement
- switch is a multi-way branching statement ‚Äî used when a variable is compared against multiple fixed values.
- üî∏ Syntax
```c
switch (expression) {
    case constant1:
        // code
        break;
    case constant2:
        // code
        break;
    default:
        // code
}
```

```c
int day = 3;

switch (day) {
    case 1: printf("Monday\n"); break;
    case 2: printf("Tuesday\n"); break;
    case 3: printf("Wednesday\n"); break;
    default: printf("Invalid day\n");
}
```

**‚öôÔ∏è Rules & Notes**
- The expression must be integral (int, char, enum).
- Each case must be a constant, not a variable or range.
- break stops execution from ‚Äúfalling through‚Äù into the next case.
- If break is omitted, the next case runs too (sometimes intentional).


**Example (fall-through behavior):**
```c
int x = 2;
switch (x) {
    case 1:
        printf("One ");
    case 2:
        printf("Two ");
    case 3:
        printf("Three ");
}
```

---
### üß© goto & Labels
- goto is a jump statement that transfers control to a labeled part of the code.
- ‚ö†Ô∏è Use with caution ‚Äî can make code hard to read, but useful for early exits or cleanup.

- Example 

```c
#include <stdio.h>
int main() {
    int n = 3;
    if (n < 5)
        goto small;

    printf("This won‚Äôt print.\n");
    return 0;

small:
    printf("Number is small.\n");
    return 0;
}
```

**‚úÖ When goto is useful:**
- Breaking from deeply nested loops
- Handling cleanup in error situations
```c
for (int i = 0; i < 3; i++) {
    for (int j = 0; j < 3; j++) {
        if (i == j)
            goto exit;
    }
}
exit:
printf("Exited loops\n");
```

**Example 1: Calculator**
```c

switch(op) {
```

### ‚ö†Ô∏è Edge Cases & Pitfalls
| Case                       | Description                                     | Example                                 |
| -------------------------- | ----------------------------------------------- | --------------------------------------- |
| Missing Braces             | Only first statement after `if` runs            | `if(x>0) printf("A"); printf("B");`     |
| Floating-Point Comparison  | Never use `==` with floats                      | `if(a == b)` ‚Üí may fail due to rounding |
| Missing `break`            | In `switch`, it causes fall-through             | See example above                       |
| Uninitialized Variables    | Using them in conditions causes garbage results | `if(flag)` before assigning value       |
| Assignment in Condition    | `if(x=5)` sets x to 5 and returns true          | Use `==` not `=`                        |
| Nested `if-else` confusion | Always use braces for clarity                   | Prevents ‚Äúdangling else‚Äù problem        |

# üåÄ Looping Statements in C

## üîπ What Are Loops?

Loops are control structures that let you repeat a block of code multiple times ‚Äî either for a fixed number of times or until a condition fails.

**Analogy:** Think of a loop like brushing your teeth every morning ‚Äî the same action, repeated daily, until a certain condition (say, you're done brushing) becomes false.

## üîπ Why Loops Are Needed

- To avoid code repetition
- To perform repeated tasks efficiently
- To iterate over data, numbers, or user inputs
- To build logic dynamically (like counting, searching, generating patterns)

---

## üß† 1Ô∏è‚É£ For Loops

### üî∏ What It Is

A for loop is ideal when you know exactly how many times you want to repeat something.

**Syntax:**
```c
for (initialization; condition; increment/decrement) {
    // code to execute repeatedly
}
```

**Flow:**
1. Initialization (runs once)
2. Condition check (before each loop)
3. Body executes if condition true
4. Increment/Decrement step
5. Repeat until condition becomes false

### üî∏ Example 1 ‚Äî Basic Counting
```c
for (int i = 1; i <= 5; i++) {
    printf("%d ", i);
}
```
**Output:** `1 2 3 4 5`

**Analogy:** Think of for loop as a "counting machine." You set where it starts, where it stops, and how fast it counts.

### üî∏ Example 2 ‚Äî Reverse Counting
```c
for (int i = 5; i >= 1; i--) {
    printf("%d ", i);
}
```
**Output:** `5 4 3 2 1`

### üî∏ Example 3 ‚Äî Multiple Variables
```c
for (int i = 0, j = 10; i <= j; i++, j--) {
    printf("%d %d\n", i, j);
}
```
**Output:**
```
0 10  
1 9  
2 8  
3 7  
4 6  
5 5
```

**Analogy:** Two people walking toward each other from opposite ends of a hallway.

### üî∏ Example 4 ‚Äî Skipping Values
```c
for (int i = 0; i <= 10; i += 2)
    printf("%d ", i);
```
**Output:** `0 2 4 6 8 10`

### üî∏ Example 5 ‚Äî Infinite For Loop
```c
for(;;)
    printf("Hello ");
```

A `for(;;)` loop has no condition, so it runs forever until stopped manually (break, Ctrl+C, etc.).

---

## üß© 2Ô∏è‚É£ While Loops

### üî∏ What It Is

A while loop runs as long as a condition is true ‚Äî perfect when the number of iterations is unknown beforehand.

**Syntax:**
```c
while (condition) {
    // code
}
```

### üî∏ Example 1 ‚Äî Simple Counter
```c
int i = 1;
while (i <= 5) {
    printf("%d ", i);
    i++;
}
```
**Output:** `1 2 3 4 5`

### üî∏ Example 2 ‚Äî Waiting for Condition
```c
int password = 0;
while (password != 1234) {
    printf("Enter password: ");
    scanf("%d", &password);
}
printf("Access granted!\n");
```

**Analogy:** Like a security gate that keeps asking for your password until it's correct.

### üî∏ Infinite While Loop Trap
```c
int i = 0;
while (i < 5) {
    printf("%d ", i);
    // forgot i++;
}
```

‚ö†Ô∏è **Infinite loop!** ‚Äî Because i never changes, `i < 5` is always true.

---

## üîÅ 3Ô∏è‚É£ Do-While Loops

### üî∏ What It Is

A do-while loop guarantees the loop body runs at least once, even if the condition is false initially.

**Syntax:**
```c
do {
    // code
} while (condition);
```

### üî∏ Example 1 ‚Äî Basic Demo
```c
int i = 1;
do {
    printf("%d ", i);
    i++;
} while (i <= 5);
```
**Output:** `1 2 3 4 5`

### üî∏ Example 2 ‚Äî Execute-at-least-once Behavior
```c
int num = 5;
do {
    printf("Runs once even if false!\n");
} while (num < 0);
```
**Output:** `Runs once even if false!`

**Analogy:** Like an elevator that must move at least once before checking where to stop.

---

## üß© 4Ô∏è‚É£ Nested Loops

### üî∏ What They Are

A loop inside another loop ‚Äî used for tables, grids, or patterns.

### Example 1 ‚Äî Multiplication Table
```c
for (int i = 1; i <= 3; i++) {
    for (int j = 1; j <= 3; j++) {
        printf("%d ", i * j);
    }
    printf("\n");
}
```
**Output:**
```
1 2 3
2 4 6
3 6 9
```

### Example 2 ‚Äî Star Pattern
```c
for (int i = 1; i <= 5; i++) {
    for (int j = 1; j <= i; j++) {
        printf("* ");
    }
    printf("\n");
}
```
**Output:**
```
*
* *
* * *
* * * *
* * * * *
```

**Analogy:**
- Outer loop = rows
- Inner loop = columns

---

## üö¶ 5Ô∏è‚É£ Break and Continue

### üî∏ break Statement

Used to exit a loop immediately, even if the condition is still true.

**Example 1 ‚Äî Stop at a value**
```c
for (int i = 1; i <= 10; i++) {
    if (i == 5)
        break;
    printf("%d ", i);
}
```
**Output:** `1 2 3 4`

### üî∏ continue Statement

Used to skip the rest of the loop body and jump to the next iteration.

**Example 2 ‚Äî Skip even numbers**
```c
for (int i = 1; i <= 10; i++) {
    if (i % 2 == 0)
        continue;
    printf("%d ", i);
}
```
**Output:** `1 3 5 7 9`

**Analogy:**
- `break` = "stop the ride"
- `continue` = "skip this turn, go to next"

---

## üèÅ 6Ô∏è‚É£ goto in Loops 

### üî∏ What It Is

`goto` lets you jump to a labeled part of your code ‚Äî useful in rare cases (like breaking multiple loops).

**Example:**
```c
for (int i = 1; i <= 3; i++) {
    for (int j = 1; j <= 3; j++) {
        if (i == j)
            goto exit;
        printf("%d%d ", i, j);
    }
}
exit:
printf("\nExited loops\n");
```
**Output:** `12 Exited loops`

**Caution:** Avoid overusing `goto`; it can make logic confusing.

---

## ‚öôÔ∏è 7Ô∏è‚É£ Edge Cases & Common Pitfalls

| ‚ùå Pitfall | üß† Explanation | üí° Example |
|-----------|----------------|------------|
| Forgetting increment/decrement | Causes infinite loop | `while(i < 10)` without `i++` |
| Using `;` after loop header | Creates an empty loop | `for(i=0;i<5;i++);` |
| Nested loops without braces | May execute only one statement | Always use `{}` |
| Wrong loop condition | May run zero or infinite times | `while(i <= 0)` when i starts at 1 |
| break in nested loop | Only breaks inner loop | Use goto or flags for outer break |
| Uninitialized counter | Garbage values = undefined behavior | `for(int i; i<5; i++)` (no init) |
